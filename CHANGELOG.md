## Unreleased

## v0.3.9 candidate

Product-path hardening:
- Fixed clean-Windows video handling when `ffprobe` is unavailable by falling back to FFmpeg stream diagnostics before conversion, while still reporting true no-audio and corrupt-media failures clearly.
- Added vendor-neutral Windows GPU Adapter Memory VRAM sampling for NVIDIA, AMD, and Intel where Windows exposes the counters, with Torch CUDA allocator data kept as a labeled fallback instead of silently comparing non-Torch backends against fake zero VRAM.
- Added VRAM measurement source fields to JSON, TXT, CSV, and HTML reports so unavailable telemetry, Windows counter telemetry, and Torch-only telemetry are distinguishable.
- Added Python-side runtime-only ranking for speed/memory and accuracy-aware balanced ranking when an LLM-corrected reference is imported, while labeling runtime-only ranking as not measuring transcript quality.
- Optimized exact-match edit-distance scoring so large LLM-reference scoring no longer spends minutes on identical chunks.
